Marketplace API Reference

Omnichannel marketplace API — Java 21 / Spring Boot behind a Kong gateway. Audience: developers.

Overview

Every request goes through the Kong gateway. Kong verifies the JWT and injects X-Auth-User / X-Auth-Role headers that services trust — you never talk to services directly. Public endpoints work without a token. Five services sit behind the gateway: Auth, Catalog, Order, Chat, Social.

Base URL: http://localhost:8080 (all paths below are relative to this)

Prerequisites

  • The stack running locally (see ผู้ดูแลระบบ — ./run.sh --build -d)
  • An HTTP client (curl, Postman, or similar)
  • A user account (create one in Quick Start below)

Quick Start (5 นาที)

# 1) Register a user (role BUYER)
curl -X POST http://localhost:8080/api/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"email":"you@example.com","username":"you","password":"password123"}'

# 2) Login -> copy the accessToken
curl -X POST http://localhost:8080/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"you","password":"password123"}'

# 3) Call an authenticated endpoint
curl http://localhost:8080/api/users/me \
  -H "Authorization: Bearer <accessToken>"

# 4) Browse products (public, no token)
curl http://localhost:8080/api/catalog/products

Authentication

  • Access token — a JWT (HS512, ~15 min TTL). Send it as Authorization: Bearer <token>.
  • Refresh token — set as an httpOnly cookie on login. Call POST /api/auth/refresh to rotate it and get a new access token.
  • Roles — BUYER, SELLER (ADMIN reserved). Upgrade with POST /api/users/me/become-seller, then refresh/login again to receive a token carrying the SELLER role.
  • Gateway trust — Kong verifies the token and injects X-Auth-User / X-Auth-Role; if no token is present it passes through so public endpoints still work.
  • Internal endpoints (/internal/**, /webhooks/**) are service-to-service (guarded by X-Internal-Key or a webhook token) — not for normal API clients.

Endpoints

Grouped by service. Auth badge = role required (Kong-injected). Examples use real field names from the code.

Error Codes

Errors use RFC 7807 application/problem+json:

{
  "type": "about:blank",
  "title": "Conflict",
  "status": 409,
  "detail": "out of stock: [1]",
  "instance": "/api/orders/checkout"
}
Codeความหมาย
200 / 201 / 204สำเร็จ (204 = ไม่มี body)
400คำขอไม่ถูกต้อง (validation)
401ไม่ได้ยืนยันตัวตน / token หมดอายุ
403ไม่มีสิทธิ์ (บทบาทไม่ถูก / ไม่ใช่เจ้าของ / internal key ผิด)
404ไม่พบทรัพยากร
409ขัดแย้ง — สินค้าหมด (oversell) หรือข้อมูลซ้ำ
500ข้อผิดพลาดฝั่งเซิร์ฟเวอร์

WebSocket close codes (/ws/chat)

Codeความหมาย
4401ยืนยันตัวตนล้มเหลว (เฟรมแรกไม่ใช่ auth หรือ token ไม่ถูก)
4403ไม่ใช่คู่สนทนาของห้องนั้น
