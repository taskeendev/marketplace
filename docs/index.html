Marketplace — เอกสารการใช้งาน

มาร์เก็ตเพลสแบบ omnichannel (ขายได้ทั้งบนเว็บและ Facebook) สร้างด้วยสถาปัตยกรรม microservices — Java 21 / Spring Boot, React, และ Kong gateway

เอกสารนี้แยกตาม บทบาทของผู้ใช้ — เลือกหน้าที่ตรงกับสิ่งที่คุณต้องทำ

ผู้ดูแลระบบ

รัน stack ด้วยคำสั่งเดียว · สถาปัตยกรรม · smoke test · ตัวแปร config

ระบบทำอะไรได้

  • ผู้ซื้อ: สมัคร/เข้าสู่ระบบ, ค้นหาสินค้า, ตะกร้า, สั่งซื้อ (ตัดสต็อกจริง กัน oversell), ดูออเดอร์, แชตเรียลไทม์กับผู้ขาย
  • ผู้ขาย: เปิดร้าน, จัดการสินค้า, จัดการออเดอร์ + สถานะ, กล่องแชตรวม (เว็บ + Facebook), เชื่อมเพจ + sync สินค้าไป FB
  • Omnichannel: ข้อความจาก Facebook เด้งเข้ากล่องแชตผู้ขาย และตอบกลับออกไปได้ (ปัจจุบัน mock Meta — เสียบของจริงเมื่อผ่าน Business verification)
